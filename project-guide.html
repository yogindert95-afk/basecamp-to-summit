01

File structure

No build tools, no bundler, no Node.js. Every file is hand-written and served directly.

basecamp-to-summit/ ├── index.html ← Home page ├── the-trailhead.html ← Campus-to-corporate program ├── base-camp.html ← Aspiring + new managers (merged) ├── the-summit.html ← Senior leaders program ├── the-ascent.html ← Redirect stub → base-camp.html ├── about.html ← About Yogi + career timeline ├── book-a-call.html ← Contact form (mailto-based) ├── css/ │ └── styles.css ← Entire design system (247 lines) ├── js/ │ └── site.js ← Shared behaviour (94 lines) ├── assets/ │ └── yogi.jpg ← Headshot photo (800×800) ├── .gitignore └── .github/workflows/ └── deploy.yml ← GitHub Pages auto-deploy

  /* backgrounds */

These CSS classes are used across multiple pages. All defined in css/styles.css.

07

Animations and interactions

All animation logic lives in js/site.js and respects prefers-reduced-motion.

Scroll reveal

Any element with class .reveal fades up into view when it enters the viewport. Uses IntersectionObserver with threshold 0.15. Elements start at opacity:0; translateY(22px) and transition to full visibility.

Trail connectors

The dashed vertical lines between curriculum legs (.leg-conn) start at scaleY(0) and draw downward when they enter the viewport. Threshold 0.4.

Stat count-up

Elements with .stat-num[data-to] count up from 0 to the target number with cubic easing over 1.2 seconds when they scroll into view.

<!-- Example: counts to 1,600 with "+" suffix -->
<div class="stat-num" data-to="1600" data-suffix="+">0</div>

Snowfall

Any element with class .snowfield gets 14 animated particle dots that drift downward continuously. Used on the pine method band on the home page.

Scroll progress bar

A thin orange gradient bar across the top of the page that fills as you scroll down. Created dynamically by site.js.

Shuffle grid (Trailhead page only)

The hero on the-trailhead.html features a 4×4 grid of stock photo tiles that reshuffle every 3 seconds using the FLIP animation technique (First, Last, Invert, Play). The animation uses spring easing and pauses when the page is hidden or the grid is out of view. This code is inline in the Trailhead HTML file.

Career timeline (About page)

An SVG polyline that draws itself via SMIL <animate> over 6.5 seconds, revealing career stops from BYJU's to BlackRock. The current position pulses with @keyframes pulseRing.

Home page hero

The hero card expands from a centered card to a fullscreen section as you scroll, with a parallax mountain scene containing an animated trekker (bobbing) and flag (waving), drifting clouds, and twinkling stars.

How to edit

Preview locally

Open a terminal, navigate to the project folder, and start any static file server:

cd ~/Documents/Claude/Finance/basecamp-to-summit
python3 -m http.server 8000
# Open http://localhost:8000 in your browser

Change the color scheme

Edit the CSS variables in css/styles.css lines 2–14. Every color throughout the site references these variables, so a single change propagates everywhere.

Change text content

Open the relevant HTML file and edit the text directly. No templating engine — what you see is what gets served. Key content locations:

  • Taglines and headings: Look for <h1>, <h2>, and .kick elements
  • Program descriptions: Inside <details class="stage"> elements
  • Stats/numbers: Find .stat-num and update both the visible text and the data-to attribute
  • Contact email: Change CONTACT_EMAIL in book-a-call.html line 85

Change the headshot photo

Replace assets/yogi.jpg with a new image. Keep the filename the same, or update the <img src> in about.html line 48. Square or portrait aspect ratio works best.

Add a new program page

  1. Copy the-summit.html as a starting template
  2. Update the nav, hero, curriculum legs, FAQ, and footer
  3. Add a link to the nav in every HTML file
  4. Add the route card to the home page's "Choose your climb" section

Change career timeline

The timeline is an inline SVG in about.html (lines 69–96). Each career stop is a <circle> + two <text> elements positioned by cx/cy coordinates. The experience cards below (lines 108–129) should match.

Update stock photos

The route cards on the home page and the shuffle grid on the Trailhead page use Unsplash URLs directly in <img src> and CSS background-image. Replace URLs to change photos. For reliability, download images to assets/ and use local paths.

⚠
Remember: If you add a new nav link, update it in all 6 page files plus the footer in each. There's no shared template — each page has its own copy of the nav and footer.

Deployment

The site auto-deploys to GitHub Pages whenever you push to the main branch.

How it works

  1. You push changes to the main branch on GitHub
  2. The GitHub Actions workflow (.github/workflows/deploy.yml) triggers automatically
  3. It checks out the code, packages it, and deploys to GitHub Pages
  4. The site is live at yogindert95-afk.github.io/basecamp-to-summit within ~30 seconds

Push changes

cd ~/Documents/Claude/Finance/basecamp-to-summit

# Stage your changes
git add -A

# Commit with a description
git commit -m "Update homepage headline"

# Push to deploy
git push

First-time setup on a new machine

# 1. Install GitHub CLI
brew install gh
# Or download from: https://github.com/cli/cli/releases

# 2. Authenticate
gh auth login --web --git-protocol https

# 3. Clone the repo
gh repo clone yogindert95-afk/basecamp-to-summit

# 4. Configure git credentials
gh auth setup-git

Custom domain (optional)

To use a custom domain like basecamptosummit.com instead of the GitHub Pages URL:

  1. Buy a domain from any registrar (Namecheap, Google Domains, Cloudflare, etc.)
  2. In your domain's DNS settings, add these records: 
    # A records (point to GitHub Pages)
A     @    185.199.108.153
A     @    185.199.109.153
A     @    185.199.110.153
A     @    185.199.111.153

# CNAME for www subdomain
CNAME www  yogindert95-afk.github.io
  3. In your repo on GitHub: Settings → Pages → Custom domain → enter your domain
  4. Check "Enforce HTTPS"
  5. Add a CNAME file to the repo root containing just the domain name: 
    basecamptosummit.com
ⓘ
DNS propagation takes 10 minutes to 48 hours. GitHub will show a warning until it verifies the domain. HTTPS is free and automatic via GitHub.
